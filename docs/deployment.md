# Развертывание проекта "Коридор" на GitHub Pages

## Автоматическое развертывание

GitHub Pages автоматически разворачивает сайты из основной ветки (main/master) репозитория. Процесс выглядит следующим образом:

1. При пуше в основную ветку GitHub автоматически запускает процесс сборки
2. Сайт собирается и публикуется в папке `gh-pages`
3. Готовый сайт становится доступен по адресу: `https://[username].github.io/[repository-name]`

## Настройка репозитория

### Основные настройки

1. В настройках репозитория включите GitHub Pages:
   - Перейдите в Settings → Pages
   - Выберите источник развертывания: Deploy from a branch
   - Выберите ветку: main
   - Укажите папку: / (root)

### Настройка кастомного домена (опционально)

Для использования собственного домена с GitHub Pages:

1. Настройте DNS вашего домена на GitHub:
   - Создайте CNAME файл с вашим доменом
   - Добавьте A-запись на ваш домен в DNS

2. В настройках вашего доменного регистратора:
   - Добавьте A-запись на GitHub Pages
   - Настройте ALIAS или CNAME для указания на GitHub Pages

## Процесс развертывания

### Локальное тестирование

Перед пушем изменений проверьте:

1. **Функциональность**: Убедитесь, что все интерактивные элементы работают
2. **Производительность**: Проверьте скорость загрузки и анимации
3. **Адаптивность**: Проверьте отображение на мобильных устройствах
4. **Кроссбраузерность**: Протестируйте в Chrome, Firefox, Safari, Edge

### Публикация

1. Сделайте коммит с описанием изменений
2. Отправьте изменения в основную ветку
3. Дождитесь завершения автоматического развертывания (обычно 1-5 минут)

## Мониторинг после развертывания

### Инструменты

- **GitHub Pages**: Встроенная аналитика GitHub
- **Google Search Console**: Отслеживание индексации
- **Lighthouse**: Регулярные аудиты производительности
- **PageSpeed Insights**: Мониторинг скорости загрузки

## Решение проблем

### Частые проблемы

1. **Сайт не отображается**: Проверьте настройки Pages и ждите повторного развертывания
2. **Ресурсы не загружаются**: Убедитесь в правильных путях в репозитории
3. **Анимации не работают**: Проверьте консоль браузера на ошибки JavaScript

### Поддержка

Если возникнут проблемы с развертыванием, создайте issue в репозитории с описанием:
- Окружение (браузер, ОС, устройство)
- Скриншот ошибки
- Ожидаемое поведение
- Шаги для воспроизведения

## Обновление контента

### Процесс

1. Внесите изменения в соответствующие файлы
2. Сделайте коммит
3. Отправьте в основную ветку
4. Дождитесь автоматического развертывания
